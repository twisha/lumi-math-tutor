What We Built

Problem → Solution

⚠️

The Problem

Young children are still learning to type. Traditional chat tutors are inaccessible to kids who need voice, patience, and verified answers — not hallucinated math.

✨

The Solution

Lumi listens via local Whisper, reasons with Claude via the Anthropic API, verifies answers through MCP tools, speaks back via local macOS TTS. A real agentic AI loop — end to end.

🎓

Two Grade Levels

K–2: counting, addition & subtraction within 20, with emoji + hop-path visual aids. Grade 3–5: multiplication, division, fractions, multi-step problems.

🛡️

Guardrails

Every input is classified for topic and grade level before responding. Out-of-scope requests get a warm redirect. Answers always tool-verified — never guessed.

AI Engineering Fundamentals

8 Course Concepts Applied

✏️

Prompt Engineering

Two grade-level personas in natural language — K–2 and 3–5. Teaching strategy, hint escalation, and guardrails. Zero fine-tuning.

🔧

Agentic Tool Use

Claude autonomously reasons when to call calculate(), check_answer(), generate_problem(), check_topic(), or check_grade_level().

🔌

MCP Protocol

Tools run as a real MCP server over stdio — the same protocol Claude Code uses. Model-agnostic: swap the LLM, keep the tools.

🏷️

LLM Self-Tagging

Lumi embeds [COUNT:N] in its own reply. App strips the tag, sets recording window to N+3 s. The model classifies its own intent — no regex.

⚡

Prompt Caching

System prompt marked cache_control: ephemeral on every API call — cuts latency and token cost on every multi-turn turn.

🎙️

Multimodal I/O

Whisper transformer STT with VAD silence-detection early-stop. macOS say TTS with Unicode normalization and 500 ms audio flush buffer.

🛡️

Guardrails

Every response topic- and grade-classified before answering. Tool-verified math — Lumi never guesses. Out-of-scope gets a warm redirect, not an error.

📊

Observability

LangSmith wraps every ask_lumi() call — full trace of tool calls, token count, latency, and conversation context.

K–2 Feature Highlight

Context-Aware Visual Aids

🎨

Hop-Path Number Line

Replaces a tiny 0–20 ruler with large 56px circles connected by arrows. ★ START marks the larger operand; yellow circles show each hop; a dashed ? marks where the child must land — answer intentionally hidden.

♿

Color-Blind Accessible

Color is never the only signal. Start circle is larger with a ★ icon and "START" label. Answer slot has a dashed border and "YOU!" label. Any child can follow the path without relying on color.

🧠

Shown Only When Struggling

Visual is hidden on first problem presentation. It appears only when the child asks for help or gives a wrong answer — detected via the child's words and check_answer() tool result.

🔗

Prompt ↔ Visual Alignment

Lumi's spoken hint references the same shapes: "Look at the ★ START circle — hop along each circle and tell me what number you land on at the ? circle." Visual and voice teach the same strategy.

13! 👦
→ check_answer(expected=13, given=13) → correct: true ✓
